Link Connect Game 
I. Project Background 
Link Connect is a classic casual game where the core gameplay involves connecting identical patterns with no more than three straight lines to eliminate them. This project requires development using the Java programming language to implement complete game logic, a graphical user interface (GUI), a user system, a save/load mechanism, and extended features, while balancing basic gameplay and playability expansion. 

II. Overall Functional Framework 
The project is divided into six core tasks based on scoring modules, with the overall functional structure as follows: 
1. 
Game Initialization and Level System (10 points) 

2. 
User System (Guest/Registration/Login/Game Records) (15 points) 

3. 
Save and Load System (15 points)

Core Game Logic (Connection, Elimination, Scoring, Win/Loss Judgment) (25 points) 

5. 
Graphical User Interface (GUI) (15 points) 

6. 
Advanced Extended Features (Self-selected implementation, counted as bonus points) (20+10 points)

IV. Game Rule Requirements 
4.1 Basic Gameplay 

Players
 click two pattern cards in sequence 


Identical
 patterns + valid connection    eliminate and score points 


Di.erent
 patterns or invalid connection    cancel selection, no points deducted 


Support
 continuous quick eliminations to form combos 



4.2 Elimination Conditions 
1. 
Two patterns are exactly the same 

2. 
The connection path has a maximum of three straight lines (maximum two turns) 

3. 
No cards block the connection space 



4.3 Win/Loss Rules 

Victory: Clear all patterns within the time limit 

Failure: Time expires without clearing all patterns OR no available eliminable pairs on the board 

Support restarting or returning to the menu after failure 
V. Team Rules 
1. Grouping Requirements 
This course project is completed in 2-person teams. All students must participate in project development and on-site defense in 2-person teams; single-person/multi-person teams are not accepted in principle. 
2. Default and Adjustment Rules for Contribution Ratio 

During
 the team defense, all team members must be present, and team members discuss and determine the initial contribution ratio. 


Review
 teachers will verify the authenticity of the initial contribution ratio based on on-site defense performance, actual development contributions, etc.; if it does not match the actual contribution, the contribution ratio will be adjusted directly, and the .nal ratio shall be subject to the teacher's approval. 


If
 a member is absent from the defense, they will be scored 0 points directly and not participate in scoring; the contribution ratio of the attending teammate shall not exceed 50%. 


The
 maximum individual score for non-presentation teams is 100 points, and presentation teams will receive supplementary points according to rules.
